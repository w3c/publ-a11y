This document provides techniques for meeting the guidelines of the Accessibility Metadata Display Guide for Digital Publications 2.1. It provides practical examples on how to extract information from an ONIX record and show it to end users.

Introduction

ONIX metadata

ONIX messages describe products for the global book supply chain and will be sent from publisher or creator of the EPUB or digital books with full audio to those who will make the products available for sale, lending or subscription. These platforms may not yet have the actual files, as they may not yet be ready, or may only choose to list them for sale if they have certain accessibility features. ONIX also only describes a product, it cannot describe the features of the reading systems on which a product may be accessed. It is important to use ONIX metadata as a complement to the accessibility data embedded within the publication itself, if describing accessible books, books with full audio and related products for the global book supply chain. An ONIX file can be used to display accessibility information in advance of publication or when you do not have access to the metadata in the digital file itself. Some accessibility information may only be available when you have access to the file itself. If you are unfamiliar with ONIX, then there is more documentation available from EDItEUR.org.

It is important to note that ONIX 3.0 includes a number of new accessibility metadata codes, some of which may not be expressible in earlier versions of ONIX.

Techniques for displaying accessibility metadata in other metadata formats can be found in the Display Techniques for Displaying Accessibility Metadata

Implementation conventions

Heading structure

The algorithms defined in this document do not include instructions for outputting their respective headings because if a given technique does not result in any information (i.e., would output a statement that no information is available), the section may be omitted (i.e., no heading would be displayed). Whether there is any information to display is only known after an algorithm is processed, which would lead to the algorithms having to include instructions to remove headings.

The headings structure to use for the techniques is defined in the guidelines but is repeated here to help guide implementations. The headings are linked to the algorithm they are associated with.

  1. "Ways of reading"
  2. "Conformance"
  3. "Navigation"
  4. "Rich content"
  5. "Hazards"
  6. "Accessibility summary"
  7. "Legal considerations"
  8. "Additional accessibility information"

Coding conventions

The code conventions used in the provided code snippet follow a structure commonly found in programming languages like Python, Java, or C++. Here's an explanation of the conventions:

Conditionals and Control Flow:
IF, ELSE IF, and ELSE (in bold and capital letters) statements are used to define different conditions and the corresponding actions to be taken.
Operators:
Written operators (is present) are used to check if a particular code or codelist is present (or not) in the metadata record.
Logical operators:
Logical operators (AND, OR, NOT, in bold and capital letters) are used to combine conditions.
String Literals:
String literals are used to represent the text that should be displayed when a particular condition is met.
Variable Naming:
The terms like "property" and "value" are used in a way that suggests a variable or data structure representing the metadata record.
Indentation:
The code uses consistent indentation to define blocks of code within conditional statements. This is crucial for readability and maintaining a clear structure.
Readability:
The code is written in a way that is intended to be easily readable and understandable even by non-coders.

Node selectors

These techniques use XPath 1.0 expressions [[xpath-10]] to verify the presence of accessibility metadata in an ONIX record and to select information from it. Some of the selectors can be optimized for processors that support newer versions of XPath (e.g., to match an element against a sequence of values rather than repeat the element for each possible value).

Translation of these XPath selectors may be necessary if the input record is transformed into another form, such as a JSON structure.

Output statements

The outputs provided in this document demonstrate the compact strings to display. For every text string, however, both a compact and descriptive option is available. These strings are provided in a JSON structure, as shown in the following example.

"ways-of-reading": {
   …
   "ways-of-reading-visual-adjustments-modifiable": {
      "compact": "Appearance can be modified",
      "descriptive": " Appearance of the text and page layout can be modified according to the capabilities of the reading system (font family and font size, spaces between paragraphs, sentences, words, and letters, as well as color of background and text)",
   },
   …
},

This document does not set a default output. Either the compact or descriptive statements can be presented to the end user, or they can even be provided the option to choose their own preference.

At the end of each output statement is an ID in brackets. This ID corresponds to a key in the JSON file containing both the compact and descriptive display statements (see the preceding example). To use the descriptive statement instead of the compact, simply select the "descriptive" subkey instead of the "compact" one associated with the ID.

The canonical English JSON file containing the compact and descriptive strings is available at https://github.com/w3c/publ-a11y/tree/main/a11y-meta-display-guide/2.1/draft/localizations/en-US.

Common Functions

In this section we define the functions common to all techniques, which are called by them during execution.

Preprocessing

Before working directly with the metadata we must read the ONIX record. This is a common starting point for all techniques that allows us to query the record directly.

This algorithm does not describe how the ONIX record is discovered and obtained.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

To generate the internal representation, run the following steps:

  • LET record be textual representation of the ONIX record for the publication given onix_record_as_text.
  • LET onix be the DOM tree that results from parsing record using an XML DOM parser.
  • return onix.

The XML file being processed may contain namespaces, in the techniques when we use XPATH we will assume that the default namespace of the root record is used.

Check for node

Many of the techniques rely on checking for the presence or absence of metadata in the ONIX record.

This algorithm takes:

  • the onix argument: the internal representation of ONIX record (output of preprocessing);
  • the path argument: the XPATH of the node to check the presence of.

To check for node, run the following steps:

  1. IF onix contains path: THEN return True.
  2. ELSE return False.

Techniques

Ways of reading

This technique relates to the Ways of reading accessibility display field.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Visual adjustments

This technique relates to the Visual adjustments accessibility display field.

Understanding the variables

all_textual_content_can_be_modified

If true, it indicates that code 36 of codelist 196 (All textual content can be modified) is present in the ONIX record.

All textual content can be modified means that the digital publication does not restrict the ability of users to modify and reflow the display of any textual content to the full extent allowed by the reading system (i.e. to change the text size or typeface, line height and word spacing, colors).

is_fixed_layout

If true, it indicates that code E201 of codelist 175 (Fixed format) is present in the ONIX record.

Fixed format means that digital publication is in fixed format (e.g. EPUB Fixed Layout).

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET all_textual_content_can_be_modified be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "36"].
  • LET is_fixed_layout be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormDetail[normalize-space() = "E201"] AND NOT the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormDetail[normalize-space() = "E200"].

Instructions

  1. IF all_textual_content_can_be_modified: THEN display "Appearance can be modified".
  2. ELSE IF is_fixed_layout: THEN display "Appearance cannot be modified".
  3. ELSE display "No information about appearance modifiability is available".

Supports nonvisual reading

This technique relates to the Supports nonvisual reading accessibility display field.

Understanding the variables

all_necessary_content_textual

If true, it indicates that code 52 of codelist 196 (All non-decorative content supports reading without sight) is present in the ONIX record.

All non-decorative content supports reading without sight means that all contents of the digital publication necessary to use and understanding, including text, images (via their alternative descriptions), audio and video material (via their transcripts, descriptions, captions or subtitles) are fully accessible via suitable reading devices, for example text-to-speech screen readers or tactile reading devices (‘Braille displays’), and nothing in the digital publication prevents or blocks the use of alternative reading modes. The entire publication can be navigated and ‘read’ using only text rendered via sound or touch, and does not require visual perception.

audio_only_content

If true, it indicates that code 01 of codelist 81 (Audiobook) is present in the ONIX record.

This means that the primary content is an Audio recording of a reading of a book or other text.

real_text

If true, it indicates that code 10 of codelist 81 (Text) is present in the ONIX record.

Text means that digital publication contains "real" text (user-selectable) as its main content (or as secondary content).

textual_alternatives

If true, it indicates that at least one of the following is present in the ONIX record:

  • code 14 of codelist 196 (Short alternative textual descriptions);
  • code 15 of codelist 196 (Full alternative textual descriptions);
  • code 16 of codelist 196 (Visualized data also available as non-graphical data);
  • code V212 of codelist 175 (Transcript);

This means that there are textual alternatives for the non-textual content.

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET all_necessary_content_textual be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "52"].
  • LET audio_only_content be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail[PrimaryContentType = "01" or ProductContentType = "01"] .
  • LET real_text be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail[PrimaryContentType = "10" or ProductContentType = "10"].
  • LET textual_alternatives be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and (ProductFormFeatureValue = "14" or ProductFormFeatureValue = "15" or ProductFormFeatureValue = "16")] OR calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormDetail[normalize-space() = "V212"].

Instructions

  1. IF all_necessary_content_textual: THEN display "Readable in read aloud or dynamic braille".
  2. ELSE IF real_text OR textual_alternatives: THEN display "Not fully readable in read aloud or dynamic braille".
  3. ELSE IF audio_only_content: THEN display "Not readable in read aloud or dynamic braille".
  4. ELSE display "No information about nonvisual reading is available".
  5. IF textual_alternatives: THEN display "Has alternative text".

Prerecorded audio

This technique relates to the Pre-recorded audio accessibility display field.

Understanding the variables

all_content_audio

If true, it indicates that code 39 of codelist 196 (Supplementary material to an audiobook is accessible) is present in the ONIX record.

This means that all supplementary visual or textual material necessary for understanding of an audiobook, is available as pre-recorded audio, or has full alternative text that can be read via text-to- speech. Only for use in ONIX 3.0 or later.

all_content_pre_recorded

If true, it indicates that code 51 of codelist 196 (All non-decorative content supports reading via pre-recorded audio) is present in the ONIX record.

This means that all contents of the digital publication necessary to use and understanding, including any text, images (via alternative descriptions), video (via audio description) is fully accessible via suitable audio reproduction. The entire publication can be navigated and ‘read’ using only pre-recorded sound, and does not require visual or tactile perception.

audiobook

If true, it indicates that code 01 of codelist 81 (Audiobook) is present in the ONIX record.

This means that the primary content is an Audio recording of a reading of a book or other text.

non_textual_content_audio

If true, it indicates that one of the following codes is present in the ONIX record:

  • code 21 of codelist 81 (Partial performance – spoken word);
  • code 22 of codelist 81 (Additional audio content not part of main content);

This indicates that pre-recorded audio content is included as part of the work. It can be Audio recording of a reading, performance or dramatization of part of the work or additional pre-recorded audio of any supplementary material such as full or partial reading, lecture, performance, dramatization, interview, background documentary or other audio content not included in the primary or un-enhanced version

non_textual_content_audio_in_video

If true, it indicates that one of the following codes is present in the ONIX record:

  • code 06 of codelist 81 (Video);
  • code 25 of codelist 81 (Narrative animation);
  • code 26 of codelist 81 (Video recording of a reading);
  • code 27 of codelist 81 (Performance – visual);
  • code 28 of codelist 81 (Other video);
  • code 29 of codelist 81 (Partial performance – video);
  • code 30 of codelist 81 (Additional video content not part of main work);

This indicates that pre-recorded video including audio content is included as part of the work. It can be video recording of a reading, performance of part of the work or additional pre-recorded video of any supplementary material such as full or partial reading, lecture, performance, dramatization, interview, background documentary or other video containing audio content.

synchronised_pre_recorded_audio

If true, it indicates that one of the following codes is present in the ONIX record:

  • code 20 of codelist 196 (Synchronised pre-recorded audio)
  • code A305 of codelist 175 (Synchronised audio)

This indicates that text-synchronised pre-recorded audio narration (natural or synthesized voice) is included for substantially all textual matter, including all alternative descriptions, e.g. via a SMIL media overlay.

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET all_content_audio be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "39"].
  • LET all_content_pre_recorded be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "51"].
  • LET audiobook be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail[PrimaryContentType = "01" or ProductContentType = "01"] .
  • LET non_textual_content_audio be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail[PrimaryContentType = "21" or ProductContentType = "22" or ProductContentType = "21" or ProductContentType = "22"] .
  • LET non_textual_content_audio_in_video be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail[PrimaryContentType = "06" or PrimaryContentType = "25" or PrimaryContentType = "26" or PrimaryContentType = "27" or PrimaryContentType = "28" or PrimaryContentType = "29" or PrimaryContentType = "30" or ProductContentType = "06" or ProductContentType = "25" or ProductContentType = "26" or ProductContentType = "27" or ProductContentType = "28" or ProductContentType = "29" or ProductContentType = "30"] .
  • LET synchronised_pre_recorded_audio be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "20"] AND calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormDetail[normalize-space() = "A305"].

Instructions

  1. IF all_content_audio AND NOT synchronised_pre_recorded_audio: THEN display "Prerecorded audio only".
  2. ELSE IF audiobook OR non_textual_content_audio OR non_textual_content_audio_in_video AND NOT all_content_pre_recorded: THEN display "Prerecorded audio clips".
  3. ELSE IF all_content_pre_recorded AND synchronised_pre_recorded_audio: THEN display "Prerecorded audio synchronized with text".
  4. ELSE either omit this display field if metadata is missing or display "No information about prerecorded audio is available".

Conformance

This technique relates to the Conformance accessibility display field.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Understanding the variables

certifier

Returns the description of code 90 of codelist 196 (Compliance certification by (name)) if present in the ONIX record.

This means that the name of the organization responsible for compliance testing and certification of the product is present.

certification_date

Returns the description of code 91 of codelist 196 (Latest accessibility assessment date) if present in the ONIX record.

This means that the date of the latest assessment or re-assessment of the accessibility of the product is present.

certifier_credentials

Returns the description of code 93 of codelist 196 (Compliance certification by (URL)) if present in the ONIX record.

This means that the the URL of a web page belonging to an organization responsible for compliance testing and certification of the product is present – typically a ‘home page’ or a page describing the certification scheme itself.

certifier_report

Returns the description of code 94 of codelist 196 (Compliance web page for detailed accessibility information) if present in the ONIX record.

This means that is present the URL of a compliance web page for detailed accessibility information. The web page should be maintained by an independent compliance scheme or testing organization. Note the web page may include information about specific national requirements or voluntary conformance reports.

epub_accessibility_10

If true, it indicates that either the code 02 of codelist 196 (EPUB Accessibility Specification 1.0 A) or the code 03 of codelist 196 (EPUB Accessibility Specification 1.0 AA) are present in the ONIX record.

This means that the publication conforms with either the requirements of EPUB Accessibility Spec 1.0, at level A or level AA.

epub_accessibility_11

If true, it indicates that code 04 of codelist 196 (EPUB Accessibility Specification 1.1) is present in the ONIX record.

This means that the publication conforms with the requirements of EPUB Accessibility Spec 1.1.

level_a

If true, it indicates that code 84 of codelist 196 (WCAG level A) or the code 02 of codelist 196 (EPUB Accessibility Specification 1.0 A) are present in the ONIX record.

This means that the publication conforms with the requirements of WCAG level A or with the requirements of EPUB Accessibility Spec 1.0 level A.

level_aa

If true, it indicates that code 85 of codelist 196 (WCAG level AA) or the code 03 of codelist 196 (EPUB Accessibility Specification 1.0 AA) are present in the ONIX record.

This means that the publication conforms with the requirements of WCAG level AA or with the requirements of EPUB Accessibility Spec 1.0 level AA.

level_aaa

If true, it indicates that code 86 of codelist 196 (WCAG level AAA) is present in the ONIX record.

This means that the publication conforms with the requirements of WCAG level AAA.

lia_compliant

If true, it indicates that code 01 of codelist 196 (LIA Compliance Scheme) is present in the ONIX record.

This means that the publication conforms with the requirements of Fondazione LIA.

wcag_20

If true, it indicates that code 80 of codelist 196 (WCAG v2.0) or the code 02 of codelist 196 (EPUB Accessibility Specification 1.0 A) or the code 03 of codelist 196 (EPUB Accessibility Specification 1.0 AA) are present in the ONIX record.

This means that the publication conforms with the requirements of WCAG version 2.0 or with the requirements of EPUB Accessibility Spec 1.0 (at level A or level AA). This is because being compliant with EPUB Accessibility 1.0 specification means at least being compliant with WCAG 2.0 specification.

wcag_21

If true, it indicates that code 81 of codelist 196 (WCAG v2.1) is present in the ONIX record.

This means that the publication conforms with the requirements of WCAG version 2.1.

wcag_22

If true, it indicates that code 82 of codelist 196 (WCAG v2.2) is present in the ONIX record.

This means that the publication conforms with the requirements of WCAG version 2.2.

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET certifier be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "90"]/ProductFormFeatureDescription.
  • LET certification_date be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "91"]/ProductFormFeatureDescription.
  • LET certifier_credentials be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "93"]/ProductFormFeatureDescription.
  • LET certifier_report be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "94"]/ProductFormFeatureDescription.
  • LET epub_accessibility_10 be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "02"] OR calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "03"].
  • LET epub_accessibility_11 be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "04"].
  • LET level_a be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "84"] OR calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "02"].
  • LET level_aa be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "85"] OR calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "03"].
  • LET level_aaa be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "86"].
  • LET lia_compliant be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "01"].
  • LET wcag_20 be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "80"] OR calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "02"] OR calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "03"].
  • LET wcag_21 be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "81"].
  • LET wcag_22 be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "82"].

Instructions

  1. IF ((epub_accessibility_10 OR epub_accessibility_11 OR wcag_20 OR wcag_21 OR wcag_22) AND (level_a OR level_aa OR level_aaa)) OR lia_compliant: THEN
    1. IF level_aaa: THEN display "This publication exceeds accepted accessibility standards".
    2. ELSE IF level_aa OR lia_compliant: THEN display "This publication meets accepted accessibility standards".
    3. ELSE IF level_a: THEN display "This publication meets minimum accessibility standards".
  2. ELSE: THEN display "No information is available".
  3. IF certifier is NOT empty: THEN
    1. display "The publication was certified by "
    2. display certifier.
  4. IF certifier_credentials is NOT empty: THEN
    1. display "The certifier's credential is"
    2. display certifier_credentials as link IF certifier_credentials is an URL.

    If the certifier metadata value is a text-based badge or a credential, then only the text is displayed. If the badge or credential is expressed as a URL, then the URL can be made a link for users to access. The display of image logos or badges is typically arranged between the certifier and the distributor. The distributor would be responsible for identifying the certifier's text or link credential and replacing it with the image or linked image.

  5. The instructions from here on define the output for the detailed conformance information. Refer to the Detailed Conformance section in the guidelines for more information on how to present these strings.

    IF epub_accessibility_10 OR epub_accessibility_11 OR wcag_20 OR wcag_21 OR wcag_22 OR level_aaa OR level_aa OR level_a: THEN display "This publication claims to meet".
  6. IF epub_accessibility_10: THEN display " EPUB Accessibility 1.0".
  7. ELSE IF epub_accessibility_11: THEN display " EPUB Accessibility 1.1".
  8. IF wcag_22: THEN display " WCAG 2.2".
  9. ELSE IF wcag_21: THEN display " WCAG 2.1".
  10. ELSE IF wcag_20: THEN display " WCAG 2.0".
  11. IF level_aaa: THEN display " Level AAA".
  12. ELSE IF level_aa: THEN display " Level AA".
  13. ELSE IF level_a: THEN display " Level A".
  14. IF certification_date: THEN
    1. display "The publication was certified on"
    2. display certification_date.
  15. IF certifier_report: THEN display "For more information refer to the certifier's report" as a named link with certifier_report as the URL used.

Navigation

This technique relates to the Navigation accessibility display field.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Understanding the variables

index_navigation

If true, it indicates that code 12 of codelist 196 (Index navigation) is present in the ONIX record.

This means that there is an index that provides direct (eg hyperlinked) access to uses of the index terms in the document body.

next_previous_structural_navigation

If true, it indicates that code 29 of codelist 196 (Next / Previous structural navigation) is present in the ONIX record.

This means that all levels of heading and other structural elements of the content are correctly marked up and (if applicable) numbered, to enable fast next heading / previous heading, next chapter / previous chapter navigation without returning to the table of contents.

page_navigation

If true, it indicates that code 41 of codelist 196 (Page list navigation) is present in the ONIX record.

This means that in the publication includes page number navigation, based on print-equivalent page numbers or digital-only static page breaks.

table_of_contents_navigation

If true, it indicates that code 11 of codelist 196 (Table of contents navigation) is present in the ONIX record.

This means that there is a Table of Contents in the publication that allows direct (eg hyperlinked) access to all levels of text organization above individual paragraphs (eg to all sections and subsections) and to all tables, figures, illustrations etc.

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET index_navigation be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "12"].
  • LET next_previous_structural_navigation be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "29"].
  • LET page_navigation be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "41"].
  • LET table_of_contents_navigation be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "11"].

Instructions

  1. IF table_of_contents_navigation OR index_navigation OR page_navigation OR next_previous_structural_navigation:
    1. IF table_of_contents_navigation: THEN display "Table of contents".
    2. IF index_navigation: THEN display "Index".
    3. IF page_navigation: THEN display "Go to page".
    4. IF next_previous_structural_navigation: THEN display "Headings".
  2. ELSE either omit this display field if metadata is missing or display "No information is available".

Rich content

This technique relates to the Rich content accessibility display field.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Understanding the variables

charts_diagrams_as_non_graphical_data

If true, it indicates that code 16 of codelist 196 (Visualized data also available as non-graphical data) is present in the ONIX record.

This means that there is a positive indication that all data presented in a visual format (graph, chart, etc) has an alternative non-graphical presentation of the same data.

chemical_formula_as_latex

If true, it indicates that code 54 of codelist 196 (Accessible chemistry content as LaTeX) is present in the ONIX record.

This means that there is a positive indication that the chemical formulae are presented using LaTeX and works with compatible assistive technology.

chemical_formula_as_mathml

If true, it indicates that code 34 of codelist 196 (Accessible chemistry content as MathML) is present in the ONIX record.

This means that there is a positive indication that the chemical formulae are presented using MathML and works with compatible assistive technology.

closed_captions

If true, it indicates that code V210 of codelist 175 (Closed captions) is present in the ONIX record.

This means that the publication contains videos with closed captions or subtitles.

described_math

If true, it indicates that code 53 of codelist 196 (Described math content) is present in the ONIX record.

This means that there is a positive indication that textual descriptions of mathematical content are present, either as alternative descriptions attached to an image of the content, or in addition to MathML or LaTeX.

full_alternative_textual_descriptions

If true, it indicates that code 15 of codelist 196 (Full alternative textual description) is present in the ONIX record.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Understanding the variables

flashing_hazard

This means that there is a positive indication that the product has a flashing hazard which must be displayed.

motion_simulation_hazard

If true, it indicates that code 17 of codelist 143 (WARNING - Motion simulation hazard) is present in the ONIX record.

This means that there is a positive indication that the product has a motion simulation hazard which must be displayed.

no_flashing_hazard

If true, it indicates that code 14 of codelist 143 (No flashing hazard warning necessary) is present in the ONIX record.

This means there is a positive indication in the ONIX record confirming there are no flashing hazards associated with this product.

no_hazards_or_warnings_confirmed

If true, it indicates that code 00 of codelist 143 (No known hazards or warnings) is present in the ONIX record.

This means there is a positive indication in the ONIX record confirming there are no associated hazard warnings with this product.

no_motion_hazard

If true, it indicates that code 18 of codelist 143 (No motion simulation hazard warning necessary) is present in the ONIX record.

This means there is a positive indication in the ONIX record confirming there are no motion simulation hazards associated with this product.

no_sound_hazard

If true, it indicates that code 16 of codelist 143 (No sound hazard warning necessary) is present in the ONIX record.

This means there is a positive indication in the ONIX record confirming there are no sound hazards associated with this product.

sound_hazard

If true, it indicates that code 15 of codelist 143 (WARNING - Sound hazard) is present in the ONIX record.

This means that there is a positive indication that the product has a sound hazard which must be displayed.

unknown_flashing_hazard

If true, it indicates that code 24 of codelist 143 (Flashing risk unknown) is present in the ONIX record.

This means the presence of flashing hazards could not be determined.

unknown_if_contains_hazards

If true, it indicates that code 08 of codelist 196 (Unknown accessibility) is present in the ONIX record.

This means that the product has not been assessed for hazards and there is no information about potential hazards.

unknown_motion_hazard

If true, it indicates that code 26 of codelist 143 (Motion simulation risk unknown) is present in the ONIX record.

This means the presence of motion simulation hazards could not be determined.

unknown_sound_hazard

If true, it indicates that code 25 of codelist 143 (Sound risk unknown) is present in the ONIX record.

This means the presence of sound hazards could not be determined.

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET flashing_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "13"].
  • LET motion_simulation_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "17"].
  • LET no_flashing_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "14"].
  • LET no_hazards_or_warnings_confirmed be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "00"].
  • LET no_motion_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "18"].
  • LET no_sound_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "16"].
  • LET sound_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "15"].
  • LET unknown_flashing_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "24"].
  • LET unknown_if_contains_hazards be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "08"].
  • LET unknown_motion_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "26"].
  • LET unknown_sound_hazard be the result of calling check for node on onix, /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "12" and ProductFormFeatureValue = "25"].

Instructions

  1. IF no_hazards_or_warnings_confirmed OR (no_flashing_hazard AND no_motion_hazard AND no_sound_hazard): THEN display "No hazards".
  2. ELSE IF unknown_if_contains_hazards OR (unknown_flashing_hazard AND unknown_motion_hazard AND unknown_sound_hazard): THEN display "The presence of hazards is unknown".
  3. ELSE IF flashing_hazard OR motion_simulation_hazard OR sound_hazard OR no_flashing_hazard OR no_motion_hazard OR no_sound_hazard OR unknown_flashing_hazard OR unknown_motion_hazard OR unknown_sound_hazard: THEN
    1. IF flashing_hazard: THEN display "Flashing content".
    2. IF motion_simulation_hazard: THEN display "Motion simulation".
    3. IF sound_hazard: THEN display "Sounds".

    5. IF unknown_motion_hazard: THEN display "Motion simulation hazards not known".
    6. IF unknown_sound_hazard: THEN display "Sound hazards not known".
    7. IF no_flashing_hazard: THEN display "No flashing hazards".
    8. IF no_motion_hazard: THEN display "No motion simulation hazards".
    9. IF no_sound_hazard: THEN display "No sound hazards".

Accessibility summary

This technique relates to the Accessibility summary accessibility display field.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Being human-written, the accessibility summary and addendum will appear in the original language of the publication. Therefore it is necessary of take care of setting up the correct language tag information.
Accessibility summary existed when no system exposed the computed accessibility metadata. With more and more systems displaying this information, most of the accessibility summary may be redundant. It may be advised to display only the accessibility addendum.

Understanding the variables

accessibility_addendum

Returns the description of code 92 of codelist 196 (Accessibility addendum) if present in the ONIX record.

This means there is a human-written text containing a short addendum to the accessibility detail of the product. It contains precise information that the publisher could not express with the other's codes.

accessibility_summary

Returns the description of code 00 of codelist 196 (Accessibility summary) if present in the ONIX record.

This means there is a human-written text containing a short explanatory summary of the accessibility of the product or the URL of a web page comprising such a summary. Summarizes the already existent information and may add information that the publisher could not express with the other codes.

known_limited_accessibility

Returns the description of code 09 of codelist 196 (Inaccessible, or known limited accessibility) if present in the ONIX record.

This means there is a human-written text containing details of and reasons for limitations on accessibility of an ebook.

lang_attribute_accessibility_addendum

Returns the lang attribute of the node containing the accessibility_addendum, or the lang attribute of the nearest ancestor.

This is the language code in which the text of the Accessibility addendum was written.

lang_attribute_accessibility_summary

Returns the lang attribute of the node containing the accessibility_summary, or the lang attribute of the nearest ancestor.

This is the language code in which the text of the Accessibility summary was written.

lang_known_limited_accessibility

Returns the lang attribute of the node containing the known_limited_accessibility, or the lang attribute of the nearest ancestor.

This is the language code in which the text of the Inaccessible, or known limited accessibility was written.

language_of_text

Returns the value of nearest code 01 of codelist 22 (Language of text) if present in the ONIX record.

This indicates the main language of the content and therefore the most probable language of accessibility summary and addendum.

publisher_contact_for_accessibility

Returns the description of code 99 of codelist 196 (Publisher contact for further accessibility information) if present in the ONIX record.

This means there is an e-mail address of the publisher to whom detailed questions about accessibility of this product may be directed.

Variables setup

  • LET onix be the result of calling preprocessing given onix_record_as_text.
  • LET accessibility_addendum be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "92"]/ProductFormFeatureDescription.
  • LET accessibility_summary be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "00"]/ProductFormFeatureDescription.
  • LET known_limited_accessibility be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "09"]/ProductFormFeatureDescription.
  • LET lang_attribute_accessibility_addendum be the value extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "92"]/ProductFormFeatureDescription/(@lang|ancestor::*/@lang)[last()].
  • LET lang_attribute_accessibility_summary be the value extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "00"]/ProductFormFeatureDescription/(@lang|ancestor::*/@lang)[last()].
  • LET lang_known_limited_accessibility be the value extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "09"]/ProductFormFeatureDescription/(@lang|ancestor::*/@lang)[last()].
  • LET language_of_text be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/Language[LanguageRole="01"]/LanguageCode.
  • LET publisher_contact_for_accessibility be the value of the node extracted from onix, using the xpath /ONIXMessage/Product/DescriptiveDetail/ProductFormFeature[ProductFormFeatureType = "09" and ProductFormFeatureValue = "99"]/ProductFormFeatureDescription.

Instructions

  1. IF lang_attribute_accessibility_addendum is NOT empty: THEN LET language_accessibility_addendum be the value of lang_attribute_accessibility_addendum.
  2. ELSE LET language_accessibility_addendum be the value of language_of_text.
  3. IF lang_known_limited_accessibility is NOT empty: THEN LET language_known_limited_accessibility be the value of lang_known_limited_accessibility.
  4. ELSE LET language_known_limited_accessibility be the value of language_of_text.
  5. IF lang_attribute_accessibility_summary is NOT empty: THEN LET language_accessibility_summary be the value of lang_attribute_accessibility_summary.
  6. ELSE LET language_accessibility_summary be the value of language_of_text.
  7. IF known_limited_accessibility is NOT empty: THEN display the content of known_limited_accessibility with language_known_limited_accessibility as language.
  8. IF accessibility_addendum is NOT empty: THEN display the content of accessibility_addendum with language_accessibility_addendum as language.
  9. ELSE IF accessibility_summary is NOT empty: THEN display the content of accessibility_summary with language_accessibility_summary as language.
  10. ELSE either omit this display field if metadata is missing or display "No information is available".
  11. IF publisher_contact_for_accessibility is NOT empty: THEN
    1. display "For more information about the accessibility of this product, please contact the publisher: "
    2. display publisher_contact_for_accessibility as link IF publisher_contact_for_accessibility is an email address.

This means that the digital product falls under European Accessibility Act exception for Fundamental modification (as defined by current regulations). eaa_exemption_micro_enterprises

If true, it indicates that code 75 of codelist 196 (EEA exception 1 – Micro-enterprises) is present in the ONIX record.

This means that the digital product falls under European Accessibility Act exemption for Micro-enterprises (as defined by current regulations). The product may not have to comply with requirements of the EAA if the publisher is a micro-enterprise.

Additional accessibility information

This technique relates to the Additional accessibility information accessibility display field.

This algorithm takes the onix_record_as_text argument: a UTF-8 string representing the ONIX record.

Understanding the variables

color_not_sole_means_of_conveying_information

If true, it indicates that code 25 of codelist 196 (Use of color is not sole means of conveying information) is present in the ONIX record.

This means that for readers with color vision deficiency, use of color (eg in diagrams, graphs and charts, in prompts or on buttons inviting a response) is not the sole means of graphical distinction or of conveying information.

dyslexia_readability

If true, it indicates that code 24 of codelist 196 (Dyslexia readability) is present in the ONIX record.

This means that the product has been specifically adapted or has features specifically to improve readability for dyslexic readers, for example specialized font, character and/or line spacing, justification and paragraph spacing, coloring and other options provided specifically to improve readability for dyslexic readers.

high_contrast_between_foreground_and_background_audio

If true, it indicates that code 27 of codelist 196 (Use of high contrast between foreground and background audio) is present in the ONIX record.

This means that foreground audio content (eg voice) is presented with no or low background noise (eg ambient sounds, music), at least 20dB below the level of the foreground, or background noise can be switched off (eg via an alternative audio track). Brief and occasional sound effects may be as loud as foreground voice so long as they are isolated from the foreground.

high_contrast_between_text_and_background

If true, it indicates that code 26 of codelist 196 (Use of high contrast between text and background color) is present in the ONIX record.

This means that body text is presented with a contrast ratio of at least 4.5:1 (or 3:1 for large/heading text).

print_equivalent_page_numbering

If true, it indicates that code 19 of codelist 196 (Print-equivalent page numbering) is present in the ONIX record.

This means that in the publication contains references to the page numbering of an equivalent printed product.

sign_language

If true, it indicates that code V213 of codelist 175 (Sign language interpretation) is present in the ONIX record.

This means that a full signing of audio content of the product supplied with the video file.

text_to_speech_hinting

If true, it indicates that code 21 of codelist 196 (Text-to-speech hinting provided) is present in the ONIX record.

This means that text-to-speech has been optimized through provision of PLS lexicons, SSML or CSS Speech synthesis hints or other speech synthesis markup languages or hinting.

ultra_high_contrast_between_text_and_background

If true, it indicates that code 37 of codelist 196 (Use of ultra-high contrast between text foreground and background) is present in the ONIX record.

This means that body text is presented with a contrast ratio of at least 7:1 (or 4.5:1 for large/heading text).

visible_page_numbering

If true, it indicates that code E205 of codelist 175 (Visible page numbering) is present in the ONIX record.

This means that the e-publication (mostly in the case of a fixed-format) contains visible page numbers.

without_background_sounds

If true, it indicates that code A312 of codelist 175 (Without background sounds) is present in the ONIX record.

This means that pre-recorded audiobook narration does not contain any background sounds, including music, sound effects, etc, though music and effects may be present if isolated from the speech (ie the sounds do not overlap).
